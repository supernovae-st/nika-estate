# The estate.yaml specification

`estate.yaml` is the per-repo provenance manifest: **every tracked file
declares what it IS**. It is GENERATED, never authored: the repo's
`scripts/estate.py` regenerates it from the tracked tree, and the drift
gate re-emits and byte-compares it on every PR. The manifest observes
before it enforces (`mode: observation`).

Two schemas exist; a repo picks by scale.

## Schema 1 · per-file rows

For small trees (~100 files: nika-registry · nika-docs). Every tracked
file gets one row:

```yaml
estate_schema: 1
mode: observation
repo: supernovae-st/nika-registry
files:
- path: "CATALOG.md"
  class: generated
  sha256: e963ad2195116af29ad1798faba59e327fc03a02aaca7a5d5d7ea0ba72e12ee2
  evidence: "in-file marker: '<!-- GENERATED by scripts/cert.py --write'"
  derivation:
    tool: "python3 scripts/cert.py --write (NIKA_BIN=nika 0.104.0)"
    gate: ".github/workflows/verify.yml step 'Certs + catalog in sync' → python3 scripts/cert.py --check"
    inputs:
    - "registry/**/*.toml"
    - "pinned artifact bytes at each entry's source.repo@source.rev:source.path"
```

Per row · `path` · `class` · `sha256` (the file's content hash) ·
`evidence` (what was read to classify it) · `derivation {tool, gate,
inputs}` for derived classes · optional `note`.

## Schema 2 · the glob estate

For large trees (thousands of files: the engine, 4000+ tracked).
Per-file rows do not scale; ordered patterns cover the tree in bulk:

```yaml
estate_schema: 2
patterns:
- glob: "crates/**/src/**"
  class: authored
  match_count: 612
  aggregate_sha256: c191153aff2ae9abc126c3115a4445d5ac5adcacb54357ad31e6cee3362c9d3e
  evidence: "the code · SPDX headers · 12-gate admission · no generation markers"
```

The rules:

- **Ordered, first-match-wins** · each tracked file is claimed by the
  first pattern that matches it, and by nothing else.
- **`files:` exceptions win** · per-file rows (singletons: a pin, a
  lockfile, an authored file inside a generated zone) take precedence
  over every pattern.
- **One aggregate hash per pattern** · `match_count` plus a single
  sha256 over the sorted `(blob-sha, path)` pairs of the pattern's
  matches: coverage drift and content drift both surface, at a
  fraction of per-file cost.
- **Coverage totality** · every tracked file must land in exactly one
  bucket. An uncovered file is exit 3, listing the paths. The last
  pattern is a `**` catchall classed `authored` + `unverified-default`:
  honesty over completeness; its matches are reported under
  `unverified:`.

## The classes

| Class | One line |
|---|---|
| `authored` | a human writes here |
| `generated` | derived by a tool from declared inputs: regenerate, never hand-edit |
| `pinned-copy` | byte-copy vendored from an upstream rev by a declared lane, integrity-gated |
| `testimonial` | impl-owned test evidence · fixtures, corpora, batteries: proves behavior; neither shipped code nor derived output |
| `foreign` | pointer to a third-party sovereign source |

Two refinements the pilots produced:

- **`authored-pin`** · a hand-meaningful pin that a bot lane advances
  PR-only, never HEAD. The derivations' INPUT (the engine's
  `SPEC_PIN`).
- **authored-with-note** · an authored file carrying a tool-maintained
  block (a status block, a bot-rewritten digest line). The class stays
  `authored`; a `note:` names the block and the lane that maintains it.

Derived classes owe a `derivation`. TESTIMONIAL and FOREIGN are leaves;
they owe nothing to the root: impl tests belong to the impl, third-party
artifacts stay sovereign at their publisher.

## The CLI contract

```sh
python3 scripts/estate.py           # default: --check
python3 scripts/estate.py --write   # regenerate estate.yaml from the tracked tree
python3 scripts/estate.py --check   # re-emit · byte-compare · report
```

| Exit | Meaning |
|---|---|
| 0 | in sync |
| 2 | unknown mode |
| 3 | coverage hole (schema 2): every uncovered path listed |
| 5 | drift: `estate.yaml` diverges from the tracked tree |

Exit 5 follows the ssot-compiler convention, deliberately distinct
from sibling gates' exit 1, so an estate divergence is distinguishable
in CI logs. **Determinism is part of the contract**: two consecutive
runs on the same tree are byte-identical.

The generator always classifies itself; the manifest never lists itself
(its sha256 cannot contain its own hash).

## The evidence discipline

Classification is **evidence-driven, never assumed**. Every `evidence`
string cites what was actually read in the tree: an in-file generation
marker, a file's own header, the workflow that rewrites it, the consumer
that proves its shape. A file with no such evidence lands in `authored`
with `note: unverified-default`, an honest default, visible in the
summary, never silently promoted.

## The staleness law

A projection may be **OLD but never WRONG**. Every derived surface
carries its source hash and derivation context; when a gate goes red,
the estate head simply does not advance. The system fails closed:
stale-but-true beats fresh-but-false.
